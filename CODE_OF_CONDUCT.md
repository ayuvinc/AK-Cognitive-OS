# Code of Conduct

## Purpose

AK Cognitive OS is a collaborative framework for building software with AI-assisted workflows. This repository expects clear, respectful, technically rigorous collaboration across issues, pull requests, reviews, discussions, and documentation.

## Our Standard

Participants are expected to:

- be respectful, direct, and professional
- keep feedback factual, specific, and work-focused
- challenge ideas with evidence rather than personal criticism
- surface risks, gaps, and disagreements clearly
- respect contributor time by keeping changes scoped and well explained

## Unacceptable Behavior

The following behavior is not acceptable:

- harassment, threats, intimidation, or abuse
- discriminatory or demeaning language
- personal attacks or repeated bad-faith escalation
- doxxing or sharing private information without permission
- spam, deliberate disruption, or intentionally misleading technical claims

## Repository-Specific Expectations

Because this repository defines agent behavior and enforcement rules, contributors must also:

- avoid weakening guardrails without clear justification
- keep documentation honest about actual framework behavior
- avoid overstating security, compliance, or automation guarantees
- raise safety concerns early when changes affect hooks, permissions, or release flow

## Enforcement

Maintainers may remove content, request changes, reject contributions, or restrict participation for conduct that violates this policy.

## Reporting

To report conduct issues, contact the repository maintainer privately. If the report also involves a security concern, follow the process in [SECURITY.md](SECURITY.md).
